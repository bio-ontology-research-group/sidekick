# Data Directory

This directory contains all intermediate and final data files generated by the SIDEKICK pipeline.

## Directory Structure

After running the complete pipeline, this directory will contain:
```
data/
├── spls/                              # Deduplicated SPL XML files (~18.5K files)
├── raw_text/                          # Preprocessed text from SPLs
├── extracted/                         # Extracted entities (indications, contraindications, side effects)
├── mapped/                            # Side effects mapped to HPO
├── hp.obo                             # HPO ontology file
├── mondo.obo                          # MONDO ontology file
├── hpo_embeddings.pkl                 # Cached HPO embeddings
├── mondo_embeddings.pkl               # Cached MONDO embeddings
└── term_mapping_cache.json            # Cached term mappings
```

## Required Input Files (Place in Root Directory)

These files must be downloaded separately and placed in the **root directory** (not in `data/`):

1. **DailyMed Archives** (5 files):
   - `dm_spl_release_human_rx_part1.zip`
   - `dm_spl_release_human_rx_part2.zip`
   - `dm_spl_release_human_rx_part3.zip`
   - `dm_spl_release_human_rx_part4.zip`
   - `dm_spl_release_human_rx_part5.zip`

2. **RxNorm Mapping File**:
   - `rxnorm_mappings.txt`

## Download Links

- **DailyMed**: https://dailymed.nlm.nih.gov/dailymed/spl-resources-all-drug-labels.cfm
- **RxNorm Mappings**: https://dailymed.nlm.nih.gov/dailymed/spl-resources-all-mapping-files.cfm
